Class GeckoJS.AclComponent


Extends GeckoJS.Component.
GeckoJS.AclComponent implements a J2EE-like access control list (ACL). In thi scheme, applications associate access privileges and authorizations with roles. Roles are arranged into groups, and each user may be a member of exactly one group. Through his group membership a user gains access privileges and authorizations assigned to the roles in that group.

Internally, user authentication and authorization information are stored in the following set of models:

 - AclUser,
 - AclGroup,
 - AclRole,
 - AclGroupRole

{Boolean} securityCheck(username, password, checkOnly, force)
Logs in a user using the provided username and password.

The password is first encrypted using MD5 before comparing it against the stored credential. If the username/password combination is valid, the user's group and roles are automatically loaded from the ACL models and stored in the application Session Singleton under the key "User". The "User" object will have the following properties:
  - username
  - password
  - description 
  - Roles
  - Groups
Parameters:
{String} username
This is the user's login
{String} password
This is the password
{Boolean} checkOnly
This flag, if true, indicates that the user shall not be logged in
{Boolean} force
This flag, if true, indicates that the user force logged.
Returns:
{Boolean} "true" if authentication succeeds; "false" otherwise
