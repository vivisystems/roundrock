Class GeckoJS.ClassRegistry


Extends GeckoJS.BaseObject, GeckoJS.Singleton.
GeckoJS.ClassRegistry provides a central Singleton registry for GeckoJS classes. Each class should be registered with its class name as the key. The registration entries are stored in the "map" field.

The class registry contains a read-only "events" field of type GeckoJS.Event. This "events" field may be associated with event listeners that are invoked whenever the repository is updated. Please see the individual method description for information on the details of the various update events.

addObject(key, obj)
Adds an object to the registry.

This method adds the given object to the registry, registering it with the give key. A "change" event is dispatched to all listeners of that event after the object has been added, where the key-object pair is passed to the event listeners as named properties of the event data object (i.e. as "eventItem.data.key" and "eventItem.data.value").

Any existing object in the registry associated with the key is replaced by the new object.

<static> GeckoJS.ClassRegistry.flush()
Flushes all objects from the registry.

This method flushes (clears) all key-object pairs stored in the registry, generating a "flush" event that is dispatched to all listeners of that event after all the key-object pairs have been flushed. This GeckoJS.ClassRegistry instance is passed to the event listeners as the event data (in the field "eventItem.data").
